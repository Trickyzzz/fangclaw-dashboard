# FangClaw 演示脚本（Phase 5 收尾）

## 90 秒版本（快速展示）

### 目标
让听众在 90 秒内理解：这是一个可解释、可追溯、可落地的投研分析系统。

### 流程
1. 打开首页（`http://127.0.0.1:3100`），一句话介绍：  
   “系统会把外部消息自动转成可执行分析，并留下完整证据链。”
2. 切到 **认知引擎**，点击 **一键体验**。
3. 展示结果区三点：
   - `MODEL` 标识（说明本次调用模型或兜底状态）
   - 公司影响与权重变化
   - `Evidence ID` 可点击
4. 点击 `Evidence ID` 进入证据详情页，强调：  
   “每次结论都可追溯到同一条证据链。”
5. 切回首页，点 **态势大屏** 收尾：  
   “它不是一次性分析，而是可持续监控。”

---

## 3 分钟版本（完整案例）

### 目标
完整走通“输入消息 -> 推理 -> 调仓建议 -> 证据追溯 -> 全局态势”。

### 流程
1. **开场（20s）**  
   首页说明主题、目标池、证据链理念。
2. **触发分析（40s）**  
   在认知引擎输入一条事件消息或直接点一键体验；展示加载过程。
3. **读结果（60s）**  
   依次讲：
   - 摘要结论与置信度
   - 六维评分/情景推演
   - 公司影响与权重变更原因
4. **证据层（40s）**  
   进入证据页，展示 source / reasoning / verification / 相关日志。
5. **态势层（20s）**  
   切态势大屏，说明系统如何持续跟踪和复盘。

---

## 演示前检查清单

1. 服务是否在线：`http://127.0.0.1:3100` 可访问。  
2. 系统是否 `realtime`：运行 `pnpm run check:demo`。  
3. 认知引擎是否能返回 `Evidence ID`。  
4. 证据页是否可打开、右侧日志是否同步出现 `ANALYSIS/WEIGHT`。  
5. 若 API 限额不足，确认页面出现兜底提示，不出现空白或死转圈。
