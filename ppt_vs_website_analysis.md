# PPT vs 网站差异分析

## PPT 核心架构（四层闭环）
1. **感知层 Sensing** - 全网痕迹捕捉（传统数据+替代数据+情绪数据）
2. **连接层 Connecting** - 实体对齐、事件去重、可信度评分、时间线拼接
3. **认知层 Cognition** - 六大因子维度（基本面/技术面/资金行为/事件驱动/情绪/替代数据）
4. **控制层 Control** - 胜率赔率、仓位建议、条件触发、行为金融干预

## PPT 核心页面
- **战场态势大屏 Dashboard Wall** - 3分钟全局概览、Top事件、风险雷达
- **个股作战卡片** - 事件时间线、六维因子雷达、情景推演、AI简评
- **行为金融干预** - 六大认知偏差、强制复核/情绪提示/冷静期

## 当前网站的问题
1. **缺少四层架构概念** - 网站只有"公司池+指标+因果分析"，没有体现感知→连接→认知→控制的闭环
2. **缺少六大因子维度** - PPT 强调六维因子（基本面/技术面/资金行为/事件驱动/情绪/替代数据），网站只有简单的"指标框架"
3. **缺少个股作战卡片** - PPT 有详细的个股页面（事件时间线+六维雷达+情景推演），网站没有
4. **缺少风险雷达** - PPT 有因子拥挤度监测、风险预警，网站没有
5. **缺少行为金融干预** - PPT 有FOMO检测、冷静期、强制复核，网站没有
6. **字体太小** - 整体字体偏小，信息密度过高但可读性差
7. **Tab 标签中英混排** - 不够清晰

## 优化方案
### 布局与字体
- 增大全局字体（标题、正文、数据）
- 简化 Tab 标签，只用中文
- 增大行高和间距

### 逻辑对齐 PPT
- 将 Tab 重命名为对应 PPT 四层架构
- 态势概览 → 改为"战场态势"（Dashboard Wall）
- 公司池 → 保留但增加六维因子雷达图
- 指标框架 → 改为"六维因子"
- 因果分析 → 改为"认知引擎"
- 变更日志 → 改为"证据链"
- 新增"风控面板"Tab
